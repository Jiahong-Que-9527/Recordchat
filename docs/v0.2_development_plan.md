## 0. Data Foundation

**Goal**: expand the current illustrative knowledge base into the minimum source
pack that basically meets the intended RecordChat scope.

### Current status

- `_staging` now exists as the manual-download workspace
- loader-facing final folders now contain an initial official + NE:ONE batch
- `_staging` is excluded from loader and ontology-graph traversal
- remaining work is concentrated in conversion, metadata, ingest verification,
  and eval coverage

### Canonical conventions

- `data/raw/_staging/` is a non-ingest download-and-normalization workspace only
- normalized files must be placed in final loader-facing folders before ingest
- governed files should carry `<filename>.<ext>.meta.json` sidecars
- required sidecar fields are `source_name`, `version`, `url`,
  `document_type`, `domain`, `registry_id`, `batch_id`, and `ingested_at`

### Canonical final destinations

- official spec docs: `data/raw/one_record_docs/spec_development/`,
  `data/raw/one_record_docs/spec_2025_07/`, and later
  `data/raw/one_record_docs/spec_2023_12/`
- official ontology: `data/raw/ontology/official/`
- official OpenAPI: `data/raw/api_specs/official/`
- official examples: `data/raw/examples/official/`
- NE:ONE docs: `data/raw/one_record_docs/ne_one/`
- NE:ONE API/config materials: `data/raw/api_specs/ne_one/`
- NE:ONE examples/tests payloads: `data/raw/examples/ne_one/`

### Required source families

- official ONE Record repo materials
- official `development` spec docs
- official `2025-07` spec docs
- official ontology files: TTL / RDF / JSON-LD
- official OpenAPI files: YAML / JSON
- official and practical JSON-LD examples
- NE:ONE repo materials: README / docs / configs / examples / tests

### Work breakdown

| Step | What to do | Main output |
|---|---|---|
| 0.1 | create raw-data staging and final directory conventions | `data/raw/*` structure aligned with `docs/data_source_plan.md` |
| 0.2 | download or clone scriptable P0 sources | official repo files, ontology files, OpenAPI files |
| 0.3 | manually download gated or awkward sources | spec pages, PDFs, developer portal files, community materials |
| 0.4 | convert HTML/PDF material to `.md` / `.txt` | ingestible text files |
| 0.5 | add sidecar metadata for every imported batch | `.meta.json` files |
| 0.6 | extract NE:ONE docs/configs/examples into ingestible files and place them in the split final folders | `data/raw/one_record_docs/ne_one/*`, `data/raw/api_specs/ne_one/*`, `data/raw/examples/ne_one/*` |
| 0.7 | re-run ingest and verify source counts/chunk mix | `/ingest` and logs |
| 0.8 | extend eval so imported sources are actually exercised | `data/eval/questions.yaml` |

### Manual download items

These should be explicitly treated as manual tasks for now:

- online spec pages from `iata-cargo.github.io`
- IATA overview pages and presentation PDFs
- OpenAPI downloads from the developer portal if not directly exposed in the repo
- hackathon / pilot / tutorial materials
- glossary and domain background curation

### Acceptance

- [x] the P0 source pack exists locally
- [x] imported files are in ingestible formats
- [x] sidecar metadata exists for each source batch
- [x] `/ingest` uses the normalized final folders rather than `_staging`
- [x] eval questions cover official docs and NE:ONE materials
